# ROOMMATES 等角投影キャラクタースプライト

Issue #6 向けに作成した、生活シミュレーションゲーム用の2Dドットキャラクター素材です。Issue #7 で採用された「南西側から北東方向を見る、等角投影風のカットアウェイ表示」に合わせています。

## キャラクター

- オータニ ハル（`otani-haru`）
- ミズハラ アオイ（`mizuhara-aoi`）
- デコピン（`navigator`）: プレイヤーから指示を受け、ゲーム内のできごとへ反映する小型の浮遊キャラクター

プレイヤー自身は画面用キャラクターとして描画せず、デコピンへ指示を出す操作主体として扱います。ゲーム画面ではプロデューサー素材を使用しません。PR #13 で生成された既存素材には、住人2体とデコピンに加えて旧プロデューサー素材も保存されています。

## 画像仕様

- フレーム: 128 × 128 px、透過PNG
- ゲーム内基準表示: 64 × 64 px
- スプライトシート: 384 × 512 px、3列 × 4行
- 足元ピボット: `(64, 118)`（フレーム左上原点）
- 足元の論理占有: 1 × 1タイル
- 配置時に確保する余白: 2 × 2タイル
- 歩行キャラの列順: `step-left`、`idle`、`step-right`
- ナビゲーターの列順: `float-a`、`idle`、`float-b`
- 行順: `south`、`east`、`north`、`west`
- 推奨歩行ループ: `step-left → idle → step-right → idle`
- 推奨浮遊ループ: `float-a → idle → float-b → idle`
- 推奨フレーム時間: 170 ms

各キャラクターのディレクトリには、以下を格納しています。

```text
<character>/
├── frames/
│   ├── south-step-left.png（歩行キャラ）
│   ├── south-float-a.png（ナビゲーター）
│   ├── south-idle.png
│   ├── south-step-right.png / south-float-b.png
│   └── ...（論理4方向 × 3コマ）
├── walk-cycle.png
└── walk-cycle-preview.gif
```

`preview-64px.png` は、ゲーム内基準の64px表示でPR #13の4素材と4方向を確認するための画像です。行順はハル、アオイ、旧プロデューサー、デコピン、列順は `south`、`east`、`north`、`west` です。デコピンはほかの素材の約85%の高さに抑えています。

## 間取り座標との対応

間取りデータは左上原点で、`x` が右、`y` が下へ増加します。等角投影画面では次のように変換します。

| 論理方向 | ワールド移動 | 画面移動 | 見た目の向き |
| --- | --- | --- | --- |
| `south` | `(0, +1)` | 左下 | 南西向き・前左3/4 |
| `east` | `(+1, 0)` | 右下 | 南東向き・前右3/4 |
| `north` | `(0, -1)` | 右上 | 北東向き・後右3/4 |
| `west` | `(-1, 0)` | 左上 | 北西向き・後左3/4 |

キャラクターのワールド座標は足元ピボットを基準にタイルへ配置し、描画順は足元の画面Y座標を基準にしてください。影は素材に含めず、レンダラー側で床面へ追加します。

## Webでの表示

ドットの輪郭を維持するため、拡大縮小時は最近傍補間を使用してください。

```css
.character-sprite {
  image-rendering: pixelated;
}
```

## 制作方針

- 等角投影の床と合う、少し上から見下ろす2.5頭身
- 暖色中心の共通パレットと濃い茶色の輪郭
- 小さい表示でも髪型・服色・持ち物で識別できる配色
- 1タイル内に収まるコンパクトな歩幅
- 特定作品のキャラクターや固有表現を複製しないオリジナルデザイン

画像は組み込み画像生成を使用し、単色クロマキー背景をローカル処理で透過PNGへ変換しています。
